## 시작하기

### 설치 및 실행
1. **EXE 실행**: `SimpleTodo.exe` 더블클릭
2. **Python 실행**: `python main.py`

### 시스템 요구사항
- **Windows**: Windows 7 이상
- **Python**: Python 3.7 이상 (소스 실행 시)
- **디스플레이**: 최소 해상도 1024x768

### 첫 화면 구성
- **상단 입력창**: 새 할일을 추가하는 영역
- **검색창**: 할일을 검색하는 영역
- **진행중 섹션**: 완료되지 않은 할일 목록
- **완료 섹션**: 완료된 할일 목록
- **분할 바**: 진행중/완료 섹션 사이의 드래그 가능한 구분선
- **최소 크기**: 300x400px

## 기본 기능

### TODO 생성
1. 상단 입력창에 할일 내용 입력
2. Enter 키 누르기 또는 추가 버튼 클릭
3. 새 할일이 "진행중" 섹션에 추가됨
4. 필요 시 납기일 설정 (선택 사항)

**팁**:
- 입력창은 여러 줄 입력 가능
- 링크 (URL, 파일 경로)도 자동 인식됨

### TODO 수정
1. TODO 아이템 더블클릭
2. 편집 다이얼로그가 열림
3. 내용, 납기일, 하위 할일, 반복 설정 수정 가능
4. "저장" 버튼 클릭하여 저장
5. "취소" 버튼 또는 ESC 키로 취소

**편집 다이얼로그 구성**:
- **내용**: 할일 내용 입력 (여러 줄 가능)
- **납기일**: 날짜 선택기로 납기일 설정
- **하위 할일**: 하위 할일 추가/수정/삭제
- **반복 설정**: 반복 할일 설정

### TODO 완료
- 각 TODO 왼쪽의 체크박스 클릭
- 완료된 TODO는 자동으로 "완료" 섹션으로 이동
- 다시 체크박스 클릭 시 "진행중" 섹션으로 복원

**반복 할일의 경우**:
- 완료 체크 시 다음 날짜의 새 할일 자동 생성
- 기존 할일은 완료 섹션으로 이동

### TODO 삭제
- 각 TODO 오른쪽의 ✕ 버튼 클릭
- 삭제 확인 없이 즉시 삭제됨
- 백업을 통해 복구 가능

### 링크 열기
- URL (`http://`, `https://`, `www.`)이 포함된 경우 자동 인식
- Windows 경로 (`C:\`, `D:\`, `\\server\`) 자동 인식
- 링크를 클릭하면 기본 브라우저 또는 탐색기로 열림

**지원 링크 형식**:
- `https://example.com`
- `www.example.com`
- `C:\Users\Documents\file.txt`
- `D:\Projects\MyProject`
- `\\server\share\folder`

## 하위 할일

### 하위 할일이란?
- 메인 할일을 더 작은 단위로 나눈 것
- 각 하위 할일마다 독립적인 내용과 납기일 설정 가능
- 체크박스로 완료 여부 관리
- 자동 정렬: **납기일 빠른 순 → 생성 시간 순**

### 하위 할일 추가
1. TODO 더블클릭하여 편집 다이얼로그 열기
2. "하위 할일" 섹션 펼치기 (▶ 클릭)
3. "+ 하위 할일 추가" 버튼 클릭
4. 새 하위 할일 입력 필드가 추가됨
5. 내용 입력 및 납기일 선택 (선택 사항)
6. "저장" 버튼 클릭

**팁**:
- 하위 할일은 개수 제한 없음
- 빈 내용의 하위 할일은 저장 시 자동 제거

### 하위 할일 표시
1. 하위 할일이 있는 TODO는 왼쪽에 **▶ 버튼** 표시
2. ▶ 버튼 클릭하면 **▼**로 변경되며 하위 할일 리스트 표시
3. 다시 클릭하면 **▶**로 변경되며 하위 할일 접힘

**진행 상태 표시**:
- TODO 오른쪽에 진행 상태 표시 (예: `2/5` - 5개 중 2개 완료)
- 모든 하위 할일 완료 시 자동으로 메인 할일도 완료 제안

### 하위 할일 관리
- **완료**: 체크박스 클릭하여 완료/미완료 전환
- **편집**: 하위 할일 더블클릭하여 내용 수정
- **삭제**: 각 하위 할일의 삭제 버튼 클릭
- **순서 변경**: 자동 정렬 (납기일 빠른 순 → 생성 시간 순)

### 하위 할일 일괄 완료
- 메인 할일 완료 시 하위 할일도 함께 완료 처리
- 반대로 메인 할일 복원 시 하위 할일도 복원

## 반복 할일

### 반복 할일이란?
- 매일, 매주, 매달 자동으로 반복되는 할일
- 완료 체크 시 다음 날짜의 새 할일 자동 생성
- 원본 할일은 완료 섹션으로 이동

### 기본 반복 설정
1. TODO 편집 다이얼로그 열기
2. "반복 설정" 섹션 펼치기 (▶ 클릭)
3. "반복" 체크박스 체크
4. 반복 빈도 선택:
   - **매일**: 매일 반복
   - **매주**: 매주 같은 요일에 반복
   - **매달**: 매달 같은 날짜에 반복
5. "저장" 버튼 클릭

**중요**: 반복 할일은 **납기일이 필수**입니다. 납기일이 없으면 반복 설정이 무시됩니다.

### 반복 할일 동작
1. 반복 할일 완료 체크
2. 다음 반복 날짜 계산:
   - **매일**: 납기일 + 1일
   - **매주**: 납기일 + 7일 (또는 선택한 다음 요일)
   - **매달**: 납기일 + 1개월 (같은 날짜)
3. 새 할일 자동 생성 (내용, 납기일 복사)
4. 원본 할일은 완료 섹션으로 이동

### 고급 옵션

#### 1. 종료일 설정
**목적**: 특정 날짜까지만 반복하도록 제한

**설정 방법**:
1. "반복 설정" 섹션에서 "종료일" 체크박스 체크
2. "날짜 선택" 버튼 클릭
3. 달력에서 종료일 선택
4. 종료일이 표시됨 (예: `2025-12-31`)

**동작**:
- 다음 반복 날짜가 종료일을 초과하면 반복 중지
- 종료일 당일까지는 반복됨
- 종료일 이후 완료 체크 시 새 할일 생성 안 됨

**예시**:
```
설정: 매일 반복, 종료일 2025-12-31
- 2025-12-30 완료 → 2025-12-31 생성 (OK)
- 2025-12-31 완료 → 생성 안 됨 (종료일 초과)
```

#### 2. 주간 반복 요일 선택
**목적**: 매주 반복 시 특정 요일에만 반복

**설정 방법**:
1. 반복 빈도를 "매주"로 선택
2. "반복 요일" 체크박스 그룹이 표시됨
3. 반복할 요일 체크 (월, 화, 수, 목, 금, 토, 일)
4. 최소 1개 이상 선택 필수

**동작**:
- 완료 체크 시 선택한 요일 중 **다음으로 가까운 요일**에 새 할일 생성
- 모든 선택 요일을 지나면 다음 주의 첫 요일로 이동

**예시 1: 평일만 반복**
```
설정: 매주, 월/화/수/목/금 선택
- 월요일 완료 → 화요일 생성
- 금요일 완료 → 다음 주 월요일 생성
```

**예시 2: 월/수/금 반복**
```
설정: 매주, 월/수/금 선택
- 월요일 완료 → 수요일 생성
- 수요일 완료 → 금요일 생성
- 금요일 완료 → 다음 주 월요일 생성
```

**팁**:
- 주말 제외하려면 월~금만 선택
- 격일 반복하려면 월/수/금 또는 화/목/토 선택

#### 3. 하위 할일 복사
**목적**: 반복 할일 완료 시 하위 할일도 함께 복사

**설정 방법**:
1. "반복 설정" 섹션에서 "반복 시 하위 할일도 함께 복사" 체크박스 체크

**동작**:
- 반복 할일 완료 시 하위 할일도 새 할일로 복사
- 하위 할일의 내용, 완료 상태, 납기일 모두 복사
- 하위 할일의 납기일도 메인 할일과 **같은 일수만큼** 조정

**예시 1: 매일 반복 + 하위 할일 복사**
```
원본 할일 (1월 1일):
- 메인: 일일 보고서 작성 (납기일: 1월 1일)
  - 하위1: 데이터 수집 (납기일: 1월 1일 10:00)
  - 하위2: 분석 (납기일: 1월 1일 14:00)

완료 후 생성된 할일 (1월 2일):
- 메인: 일일 보고서 작성 (납기일: 1월 2일)
  - 하위1: 데이터 수집 (납기일: 1월 2일 10:00)
  - 하위2: 분석 (납기일: 1월 2일 14:00)
```

**예시 2: 매주 반복 + 하위 할일 복사**
```
원본 할일 (1월 1일 월요일):
- 메인: 주간 회의 준비 (납기일: 1월 1일)
  - 하위1: 자료 준비 (납기일: 12월 31일)
  - 하위2: 발표 준비 (납기일: 1월 1일 09:00)

완료 후 생성된 할일 (1월 8일 월요일):
- 메인: 주간 회의 준비 (납기일: 1월 8일)
  - 하위1: 자료 준비 (납기일: 1월 7일)  ← 7일 추가
  - 하위2: 발표 준비 (납기일: 1월 8일 09:00)  ← 7일 추가
```

**주의사항**:
- 하위 할일의 완료 상태는 복사되지 않음 (모두 미완료로 초기화)
- 납기일이 없는 하위 할일은 그대로 복사됨 (조정 안 됨)

### 반복 설정 조합 예시

#### 조합 1: 평일 아침 운동 (종료일 있음)
```
- 반복: 매주
- 요일: 월, 화, 수, 목, 금
- 종료일: 2025-12-31
- 하위 할일 복사: 체크
  - 하위1: 준비 운동
  - 하위2: 본 운동
  - 하위3: 정리 운동
```

#### 조합 2: 월간 보고서 (하위 할일 있음)
```
- 반복: 매달
- 종료일: 없음
- 하위 할일 복사: 체크
  - 하위1: 데이터 수집 (납기일: 매월 25일)
  - 하위2: 분석 (납기일: 매월 28일)
  - 하위3: 보고서 작성 (납기일: 매월 말일)
```

#### 조합 3: 프로젝트 마일스톤 (종료일 고정)
```
- 반복: 매주
- 요일: 금
- 종료일: 2025-06-30 (프로젝트 종료일)
- 하위 할일 복사: 체크
```

## 검색 및 정렬

### 검색 기능
1. 상단 검색창에 키워드 입력
2. 실시간으로 필터링됨
3. 메인 할일 내용 + 하위 할일 내용 모두 검색

**검색 특징**:
- **대소문자 구분 없음**
- **부분 문자열 매칭**: "할일" 입력 시 "오늘 할일", "내일 할일" 모두 검색
- **하위 할일 포함**: 하위 할일에 키워드가 있어도 메인 할일 표시
- **실시간 필터링**: 입력 시마다 즉시 반영

**검색 초기화**:
- 검색창 비우기
- ESC 키 누르기

**검색 팁**:
```
"회의" → 회의 관련 모든 할일 검색
"2025" → 2025년 관련 할일 검색
"중요" → 중요 표시된 할일 검색
```

### 정렬 기능
헤더의 정렬 드롭다운에서 선택:

#### 1. 오늘 우선
- **우선순위 1**: 납기일이 오늘인 할일
- **우선순위 2**: 납기일이 과거인 할일 (만료된 것)
- **우선순위 3**: 납기일이 미래인 할일
- **우선순위 4**: 납기일이 없는 할일
- 같은 우선순위 내에서는 생성 시간 순

**추천**: 기본 정렬로 추천. 오늘 해야 할 일을 바로 볼 수 있음

#### 2. 빠른 순 (납기일 오름차순)
- 납기일이 가까운 순서대로 정렬
- 납기일 없는 할일은 맨 아래
- 같은 납기일은 생성 시간 순

**추천**: 데드라인 중심으로 일할 때

#### 3. 늦은 순 (납기일 내림차순)
- 납기일이 먼 순서대로 정렬
- 납기일 없는 할일은 맨 아래
- 같은 납기일은 생성 시간 순

**추천**: 장기 계획 확인할 때

#### 4. 수동 정렬
- 드래그앤드롭으로 직접 순서 조정
- 정렬 순서가 `order` 필드에 저장됨
- 다른 정렬 모드로 변경 후 다시 수동으로 돌아오면 저장된 순서 유지

**사용 방법**:
1. "수동" 정렬 선택
2. TODO 아이템을 드래그하여 원하는 위치로 이동
3. 순서가 자동 저장됨

**팁**:
- 수동 정렬 중에도 완료/미완료 섹션 이동은 자동
- 진행중/완료 섹션 각각 독립적으로 정렬 가능

### 정렬 순서 저장
- 선택한 정렬 방식은 자동 저장
- 앱 재시작 시에도 유지됨
- `data.json`의 `settings.sortOrder`에 저장

## 시스템 설정

### 시스템 트레이
앱 최소화 시 작업 표시줄이 아닌 시스템 트레이로 이동

**트레이 아이콘 동작**:
- **더블클릭**: 앱 표시/숨김 토글
- **우클릭**: 컨텍스트 메뉴
  - **열기**: 앱 창 표시
  - **종료**: 앱 완전 종료

**트레이 사용 시나리오**:
1. 앱 창 최소화 → 트레이로 이동
2. 작업 중 다른 프로그램 사용
3. 트레이 아이콘 더블클릭 → 앱 다시 표시

**팁**:
- 트레이에 숨겨져 있어도 데이터는 자동 저장됨
- 시스템 종료 전 트레이에서 "종료" 선택 권장

### 항상 위 설정
다른 윈도우 위에 항상 표시

**설정 방법**:
1. 메뉴: **설정 → 항상 위**
2. 체크박스 토글
3. 설정 즉시 적용

**사용 시나리오**:
- 작업하면서 할일 확인
- 웹 브라우저 옆에 TODO 앱 고정
- 화상 회의 중 할일 확인

**주의**:
- 다른 윈도우를 가려서 불편할 수 있음
- 필요 시에만 활성화 권장

### 분할 비율 조정
진행중/완료 섹션의 크기 비율 조정

**조정 방법**:
1. 두 섹션 사이의 드래그 바 (가운데 선) 찾기
2. 마우스로 드래그하여 위/아래로 이동
3. 비율이 실시간으로 변경됨
4. 마우스 버튼을 놓으면 자동 저장

**제약 사항**:
- 각 섹션 최소 10% 보장
- 진행중 섹션: 10% ~ 90%
- 완료 섹션: 10% ~ 90%

**기본 비율**:
- 진행중: 90% (9)
- 완료: 10% (1)
- 비율: [9, 1]

**추천 비율**:
- **진행중 중심**: [9, 1] - 진행중 할일에 집중
- **균등 분할**: [5, 5] - 완료 내역도 자주 확인
- **완료 중심**: [3, 7] - 완료 내역 복습

**비율 저장**:
- `data.json`의 `settings.splitRatio`에 저장
- 앱 재시작 시에도 유지됨

### 단일 인스턴스
앱은 하나만 실행 가능

**동작**:
1. 앱 최초 실행 → 정상 실행
2. 앱 재실행 시도 → 기존 창 활성화
3. 트레이에 숨겨져 있어도 기존 창 표시

**포트 사용**:
- 내부적으로 포트 **65432** 사용
- 다른 프로그램이 이 포트를 사용 중이면 실행 실패

**문제 해결**:
- 포트 충돌 시 작업 관리자에서 기존 프로세스 종료
- 또는 포트 변경 (개발자 옵션)

## 문제 해결

### Q1: 앱이 시작되지 않아요

**증상**:
- 앱 실행 시 아무 창도 나타나지 않음
- 또는 에러 메시지 표시

**해결 방법**:

1. **Python 버전 확인** (소스 실행 시)
   ```bash
   python --version
   # Python 3.7 이상이어야 함
   ```

2. **의존성 설치** (소스 실행 시)
   ```bash
   pip install -r requirements.txt
   ```

3. **로그 확인**
   - 콘솔 창의 에러 메시지 확인
   - `TodoPanel_Data/` 폴더의 로그 파일 확인

4. **포트 충돌 확인**
   - 포트 65432가 사용 중인지 확인
   - 작업 관리자에서 기존 프로세스 종료

5. **재설치** (EXE 실행 시)
   - 최신 버전 다운로드
   - 기존 EXE 삭제 후 새 버전 실행

### Q2: 데이터가 사라졌어요

**증상**:
- 앱 시작 시 모든 TODO가 사라짐
- 또는 일부 TODO만 표시됨

**해결 방법**:

1. **백업 폴더 확인**
   - 경로: `TodoPanel_Data/backups/`
   - 백업 파일: `data_backup_YYYYMMDD_HHMMSS.json`

2. **백업 파일 복구**
   ```bash
   # 1. 앱 종료
   # 2. 백업 파일 찾기 (가장 최근 날짜)
   # 3. 복사
   copy TodoPanel_Data\backups\data_backup_20250101_120000.json TodoPanel_Data\data.json
   # 4. 앱 재시작
   ```

3. **수동 데이터 확인**
   - `TodoPanel_Data/data.json` 파일 열기
   - JSON 형식 확인
   - `todos` 배열에 데이터가 있는지 확인

4. **백업 정책**
   - 앱 시작 시마다 자동 백업
   - 최대 10개 백업 파일 유지
   - 오래된 백업은 자동 삭제

### Q3: 반복 할일이 생성되지 않아요

**증상**:
- 반복 설정했는데 완료 체크 시 새 할일이 생성 안 됨

**원인 및 해결**:

1. **납기일 미설정**
   - **원인**: 반복 할일은 납기일 필수
   - **해결**: 편집 다이얼로그에서 납기일 설정

2. **종료일 초과**
   - **원인**: 다음 반복 날짜가 종료일을 넘음
   - **해결**: 종료일 연장 또는 종료일 체크박스 해제

3. **주간 반복 요일 미선택**
   - **원인**: 매주 반복인데 요일 선택 안 함
   - **해결**: 최소 1개 이상 요일 선택

4. **반복 빈도 확인**
   - 편집 다이얼로그 → 반복 설정 → 빈도 확인
   - "반복" 체크박스가 체크되어 있는지 확인

**디버깅 팁**:
```
편집 다이얼로그 열기
→ 반복 설정 섹션 펼치기
→ 다음 항목 확인:
   ✓ "반복" 체크박스 체크됨
   ✓ 납기일 설정됨
   ✓ 빈도 선택됨 (매일/매주/매달)
   ✓ 종료일 없음 또는 미래 날짜
   ✓ (매주인 경우) 요일 선택됨
```

   - 전체 TODO 다시 표시

검색어: "회의"

### Q6: 드래그앤드롭이 안 돼요

**증상**:
- TODO를 드래그해도 순서가 변경 안 됨

**원인 및 해결**:

1. **정렬 모드 확인**
   - **원인**: 수동 정렬 모드가 아님
   - **해결**: 헤더에서 "수동" 정렬 선택

2. **다른 정렬 모드**
   - 오늘 우선, 빠른 순, 늦은 순에서는 드래그 불가
   - 이 모드들은 자동 정렬이므로 수동 변경 불가

3. **섹션 간 이동**
   - 진행중 ↔ 완료 섹션 간 드래그 불가
   - 완료 체크박스로만 이동 가능

✓ https://example.com

✗ example.com (프로토콜 없음)
✗ C:\ Users\... (공백)

## 데이터 관리

### 저장 위치
```
TodoPanel_Data/
├── data.json              # 메인 데이터 파일
└── backups/              # 백업 폴더
    ├── data_backup_20250101_120000.json
    ├── data_backup_20250102_130000.json
    └── ...
```

### 데이터 구조
```json
{
  "version": "1.0",
  "settings": {
    "sortOrder": "today_first",
    "splitRatio": [9, 1],
    "alwaysOnTop": false
  },
  "todos": [
    {
      "id": "unique-id",
      "content": "할일 내용",
      "completed": false,
      "createdAt": "2025-01-01T10:00:00Z",
      "dueDate": "2025-01-05T00:00:00Z",
      "order": 0,
      "subTasks": [
        {
          "id": "sub-id",
          "content": "하위 할일",
          "completed": false,
          "createdAt": "2025-01-01T10:05:00Z",
          "dueDate": "2025-01-04T00:00:00Z"
        }
      ],
      "recurrence": {
        "enabled": true,
        "frequency": "daily",
        "endDate": "2025-12-31T00:00:00Z",
        "weekdays": [1, 3, 5],
        "copySubTasks": true
      }
    }
  ]
}
```

### 자동 저장
- **타이밍**: 데이터 변경 후 500ms 후 자동 저장
- **디바운싱**: 연속 변경 시 마지막 변경 500ms 후 저장
- **원자적 저장**: 임시 파일 생성 → 검증 → 원본 교체
- **재시도**: 저장 실패 시 최대 3회 재시도

### 백업 정책
- **생성 시점**: 앱 시작 시마다 자동 백업
- **보관 개수**: 최대 10개
- **파일명**: `data_backup_YYYYMMDD_HHMMSS.json`
- **자동 삭제**: 10개 초과 시 가장 오래된 것 삭제

### 데이터 복구
1. 앱 종료
2. `TodoPanel_Data/backups/` 폴더 열기
3. 복구할 백업 파일 선택 (날짜 확인)
4. `TodoPanel_Data/data.json`으로 복사
5. 앱 재시작

**복구 팁**:
- 파일 이름의 날짜/시간 확인
- 가장 최근 백업부터 시도
- 여러 백업 파일 비교하여 선택

## 업데이트 및 버전 관리

### 버전 확인
- 앱 창 제목에 버전 표시
- 예: `Simple ToDo v2.5.0`

### 업데이트 방법
1. 최신 버전 다운로드
2. 기존 앱 종료
3. 새 버전 실행
4. 데이터 자동 마이그레이션

**주의**:
- `TodoPanel_Data/` 폴더는 삭제하지 말 것
- 백업 파일도 유지할 것

### 레거시 마이그레이션
이전 버전의 데이터도 자동 변환:
- `text` → `content`
- `created_at` → `createdAt`
- `due_date` → `dueDate`
- `position` → `order`
